NAME

geheim - Encodes/Decodes a file by generating a key from its own content.

SYNOPSIS

geheim [options] [file...]

DESCRIPTION

geheim is a command-line tool written in C that encodes/decodes a specified file. It works seamlessly on Unix, Linux and macOS.

OPTIONS

-h, --help

Display a short help message and exit.

-v, --version

Version information and exit.

-o, --overwrite

Will encode/decode the specified filename onto itself.

-e, --encode

Encode the specified file. Omitting this option will decode the specified file.

-f, --file

Followed by the filename to be encoded/decoded.

PASSPHRASE

You will be prompted to enter a passphrase. This optional coding step will
be bypassed by not providing one (a blank return).
If a passphrase is entered during the encoding process, this same
passphrase must be entered when decoding the file.
This optional additional encryption will augment the standard encryption to
make it suitable to be encoded by specific individuals.

EXAMPLES

Encode:

geheim -e -f filename

Decode:

geheim -f filename

Encode and overwrite:

geheim -e -o -f filename

NOTES

The resulting file of an Encoding or Decoding request will have the
extention .geheim attached to it, unless the overwrtie flag (-o or
--overwrite) is set.
If the overwrite flag is set the result will be overwriting the original
content.

RETURN VALUES

0

Success.

1

A process, non-exisiting file or specified option flag(s) (omission/unknown) error. An apropriate error message will be displayed.
